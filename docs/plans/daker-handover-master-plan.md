# Daker Handover Master Plan

이 문서는 [docs/ref/hackathons/daker-handover-2026-03.md](/C:/Users/gmdqn/daykervibe/docs/ref/hackathons/daker-handover-2026-03.md)를 기준으로 한 제출물 중심 계획 문서다.

## Summary
- 준비 단위는 `기획서`, `웹 URL + GitHub`, `PDF` 3개 제출물이다.
- 제품 범위는 SSOT의 필수 구현을 먼저 충족하고, 선택 확장은 심사 대응에 유리한 범위만 추가한다.
- Expedition Hub의 핵심 확장 포인트는 `베이스캠프 + 작전실 워크플로우` 구조다.
- 평가 기준 4축 `기본 구현`, `확장`, `완성도`, `문서/설명`이 각 제출물에 모두 연결되도록 설계한다.

## submission_1_plan
### 목표
- `2026-03-30 10:00 KST` 이전에 기획서 제출
- 심사자가 서비스 가치, 페이지 구성, 시스템 구조, 제출 준비 워크플로우를 바로 이해하게 만들기

### 포함 항목
- 서비스 개요
- 페이지 구성
- 시스템 구성
- 핵심 기능 명세
- 주요 사용 흐름
- 개발 및 개선 계획

### 서사
- 문서만 남은 상태에서 해커톤 포털 경험을 Expedition Hub 관점으로 재구성
- 해커톤 탐색 -> 원정대 구성 -> 베이스캠프 요약 -> 작전실 준비 -> 결과 확인 흐름
- 더미 데이터와 `localStorage` 기반이지만 서비스로서 충분히 설득 가능한 구조

### 기획서에서 강조할 점
- 기본 구현 30점:
  - 데이터 기반 렌더링
  - 상세 섹션 완성도
  - 빈 상태 UI
- 확장 30점:
  - 베이스캠프 상태 요약
  - 작전실 워크플로우 카드 보드
  - 제출 준비 흐름 설명력
- 완성도 25점:
  - 동선
  - 안정성
  - 반응형
- 문서/설명 15점:
  - 구조도
  - 재현 방법
  - 검증 방법

1. 디자인 고도화
2. 상태/태그/기간 필터
3. 원정대 초대/수락/거절과 유의사항 팝업
4. 워크플로우 카드 보강 시드

### 제출 전 점검
- Vercel URL 외부 접속 가능
- GitHub 저장소 링크 공개 또는 심사자 접근 가능
- `localStorage` 새로고침 후 유지
- 깨진 링크 없음
- 빈 상태 / 오류 상태 확인 가능
- 모바일 기본 대응

## acceptance_checklist
- 기획서에 `GitHub 저장소 링크 필수`가 누락되지 않음
- 웹 제출에 `Vercel URL`과 `GitHub 링크`가 함께 준비됨
- PDF가 실제 구현 화면과 일치함
- 심사자가 별도 키 없이 검수 가능
- `미제출` 상태와 상태 UI 3종이 실제 동작 또는 시연 가능
- 저작권과 라이선스 위반 요소 없음

## risk_log
- 가장 큰 리스크: 기본 구현 미완성 상태에서 확장 기능 과투자
- 다음 리스크: 작전실이 에디터나 협업툴처럼 과도하게 커지는 것
- 다음 리스크: 문서와 구현 화면 불일치
- 다음 리스크: 배포 URL 불안정 또는 마감 직전 장애
- 회피 원칙:
  - SSOT 필수 구현 먼저
  - 선택 확장은 기본 구현 안정 이후
  - 워크플로우 보드는 `카드 이동 기반 상태 관리` 범위로 제한
  - 배포와 PDF는 공식 마감보다 먼저 내부 잠금
