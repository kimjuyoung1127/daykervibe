## 1. 서비스 개요
- 서비스명: `Expedition Hub`
- 한줄 소개: `여러 해커톤에서 원정대 구성부터 제출 준비 관리, 결과 확인까지 이어지는 재사용형 해커톤 실행 포털`
- 개발 목표:
  - 해커톤 참가자가 해커톤 탐색, 원정대 구성, 제출 준비를 한 흐름 안에서 수행할 수 있게 한다.
  - 원정대는 작전실에서 제출 준비 상태와 실행 순서를 함께 이해하고 관리할 수 있어야 한다.
  - 심사자는 공개 페이지와 제출 상태 요약만으로도 서비스 구조와 준비 흐름을 빠르게 검수할 수 있어야 한다.
  - 이번 DAKER 해커톤을 첫 적용 사례로 삼아, 다음 해커톤에도 재사용 가능한 구조를 제안한다.

## 2. 타깃 사용자와 기대 효과
- 참가자:
  - 해커톤 정보와 모집 중인 원정대를 탐색하는 사용자.
  - 기대효과: 흩어진 정보를 오가며 찾는 시간을 줄이고, 참여 결정이 빨라진다.
- 원정대장:
  - 원정대를 만들고 제출 준비 전체 흐름을 관리하는 사용자.
  - 기대효과: 현재 단계와 다음 해야 할 일을 한눈에 파악해 준비 누락을 줄인다.
- 원정대원:
  - 원정대에 합류하고 현재 진행 상황을 함께 이해해야 하는 사용자.
  - 기대효과: 팀 내 역할과 우선순위를 공유해 실행 순서 이해도가 높아진다.
- 심사자:
  - 공개 URL만으로 서비스 가치와 결과 흐름을 확인하는 사용자.
  - 기대효과: 별도 로그인이나 외부 키 없이 핵심 페이지, 제출 상태, 리더보드를 빠르게 검수할 수 있다.
- 운영자:
  - 다음 해커톤에도 재사용 가능한 구조를 확인하려는 주최 측.
  - 기대효과: 단일 이벤트 페이지가 아닌 운영 포털 구조로 확장 가능성을 검토할 수 있다.

## 3. 페이지 구성
### 주요 페이지와 연결 관계
- `/` → `/hackathons`, `/camp`, `/rankings`
- `/hackathons` → `/hackathons/:slug`
- `/hackathons/:slug` → `/camp?hackathon=:slug`
- `/camp` → `/war-room/:teamId`
- `/hackathons/:slug` 내부에서 `제출`과 `리더보드`를 함께 이해
- `/war-room/:teamId`는 `베이스캠프 + 작전실`이 결합된 실행 허브

### 페이지별 설명
#### `/`
- Expedition Hub 소개, 한줄 소개, 핵심 가치, 3개 진입 버튼을 보여준다.
- 사용자는 `해커톤 보기`, `원정대 찾기`, `랭킹 보기` 버튼으로 주요 흐름에 바로 진입한다.

#### `/hackathons`
- 해커톤 목록, 상태, 태그, 기간, 참가자 수를 카드형으로 보여준다.
- 사용자는 필터를 선택하거나 카드를 클릭해 원하는 해커톤 상세로 이동한다.

#### `/hackathons/:slug`
- 개요, 안내, 평가, 일정, 상금, 팀, 제출, 리더보드를 한 화면 맥락으로 보여준다.
- 사용자는 `원정대 만들기`, `원정대 찾기`, 제출 가이드 확인, 리더보드 확인을 수행할 수 있다.

#### `/camp`
- 원정대 모집 목록과 생성 폼을 보여준다.
- 사용자는 특정 해커톤 기준으로 원정대를 찾거나 새 원정대를 만들 수 있다.

#### `/rankings`
- 글로벌 랭킹과 활동 요약을 보여준다.
- 사용자는 현재 순위와 점수를 확인하고 다른 참가자와 비교할 수 있다.

#### `/war-room/:teamId`
- 상단에는 베이스캠프 요약, 하단에는 작전실 워크플로우 보드를 보여준다.
- 사용자는 현재 제출 단계, 다음 해야 할 일, 플랜 카드, 체크리스트, 제출 링크를 확인하고 수정할 수 있다.

## 4. 시스템 구성
- Expedition Hub는 `Next.js + TypeScript + Tailwind + localStorage + Vercel` 기반의 프론트엔드 중심 서비스다.
- 초기 화면은 더미 JSON 데이터를 기반으로 구성하고, 사용자의 변경 사항은 `localStorage`에 저장해 새로고침 후에도 유지한다.
- 데이터 구조는 `hackathons`, `teams`, `submissions`, `leaderboards`를 중심으로 하며, 팀 전용 상태 관리를 위해 `warRooms`, `warRoomWorkflowCards`, `warRoomChecklist`를 추가한다.
- 데이터는 두 층으로 나뉜다.
  - 공개 데이터: 해커톤 목록, 상세 정보, 공개 원정대 모집 정보, 랭킹, 제출 상태 요약
  - 팀 전용 데이터: 베이스캠프 요약, 워크플로우 카드, 체크리스트, 팀 메모, 제출 링크
- 심사자는 공개 페이지와 예시 원정대가 포함된 데모 상태를 통해 전체 흐름을 검수한다. 민감한 팀 내부 정보는 공개하지 않는다.

## 5. 핵심 기능 명세
### 원정대 생성
- 입력: 사용자가 `원정대 만들기` 버튼을 누르고 팀명, 소개, 모집 포지션, 연락 링크를 입력한다.
- 동작: 시스템이 원정대를 생성하고 선택한 해커톤과 연결한다.
- 결과: 사용자는 원정대 단위로 제출 준비를 시작할 수 있다.

### 원정대 탐색
- 입력: 사용자가 캠프 페이지에서 해커톤 필터를 선택하거나 모집 중인 원정대 카드를 조회한다.
- 동작: 시스템이 조건에 맞는 원정대 목록을 렌더링한다.
- 결과: 사용자는 참여 가능한 원정대를 빠르게 찾고 합류 대상을 결정할 수 있다.

### 해커톤 상세 이해
- 입력: 사용자가 해커톤 카드를 클릭해 상세 페이지에 진입한다.
- 동작: 시스템이 개요, 평가, 일정, 상금, 제출, 리더보드 등 8개 핵심 섹션을 표시한다.
- 결과: 사용자는 이 해커톤의 규칙과 준비 방식을 한 화면에서 이해할 수 있다.

### 베이스캠프 상태 확인
- 입력: 사용자가 작전실에 진입한다.
- 동작: 시스템이 원정대명, 연결된 해커톤, 현재 제출 단계, 다음 해야 할 일, 멤버 참여 상태를 상단에 요약 표시한다.
- 결과: 팀 전체가 현재 위치와 우선 작업을 한눈에 파악한다.

### 작전실 워크플로우 관리
- 입력: 사용자가 플랜 카드를 드래그해 `기획서 준비`, `웹 제출 준비`, `PDF 준비`, `제출 완료` 컬럼 사이에서 이동한다.
- 동작: 시스템이 카드 위치와 메모를 저장하고 현재 제출 준비 단계를 반영한다.
- 결과: 팀원들이 같은 실행 순서를 공유하며 제출 준비 과정을 시각적으로 관리할 수 있다.

### 체크리스트 및 링크 관리
- 입력: 사용자가 체크리스트 항목을 추가하거나 기획서 URL, 웹 URL, GitHub 링크, PDF 링크를 입력한다.
- 동작: 시스템이 항목 상태와 제출 자산을 저장한다.
- 결과: 실제 제출 준비 수준이 누락 없이 정리되고, 다음 제출 단계로 이어진다.

### 리더보드 확인
- 입력: 사용자가 리더보드 섹션 또는 랭킹 페이지를 연다.
- 동작: 시스템이 제출 여부와 점수 상태를 기준으로 순위를 렌더링한다.
- 결과: 참가자와 심사자 모두 결과와 현재 위치를 쉽게 해석할 수 있다.

## 6. 주요 사용 흐름
1. 사용자가 메인에서 해커톤을 찾는다.
2. 상세 페이지에서 평가, 일정, 상금, 제출 구조를 이해한다.
3. 원정대를 만들거나 기존 원정대에 합류한다.
4. 작전실에서 현재 단계와 다음 액션을 확인한다.
5. 워크플로우 카드와 링크 관리를 통해 제출 준비를 진행한다.
6. 제출 상태와 리더보드를 통해 결과를 확인한다.

## 7. 개발 및 개선 계획
- 기본 구현은 필수 라우트, 상세 8섹션, 상태 UI 3종, 제출 최소 흐름을 우선 완성한다.
- 확장 포인트는 `베이스캠프 + 작전실 워크플로우` 구조에 집중한다.
- 작전실은 편집기나 메신저처럼 확장하지 않고, 카드 이동 기반 상태 관리 허브로 유지한다.
- 모바일 대응, 빈 상태 UI, 새로고침 후 상태 유지, 비공개 정보 비노출을 완성도 항목의 핵심으로 본다.
